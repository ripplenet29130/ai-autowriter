# AutomaticWriter テスト手順書

このドキュメントは、AutomaticWriter の各機能について、確認手順と期待結果を整理したものです。  
開発時の確認、リリース前確認、外販前の受け入れ確認に利用します。

## 1. テスト実施前の共通確認

### 1.1 環境

- [ ] Node modules がインストール済み
- [ ] `.env.local` または必要な環境変数が設定済み
- [ ] AI設定が少なくとも1件有効
- [ ] WordPress設定が少なくとも1件有効
- [ ] Supabase 接続情報が正しい
- [ ] Perplexity API Key を使う場合は設定済み

### 1.2 事前コマンド

1. `npx tsc -b`
2. `npm run dev`
3. 可能なら `npm run build`

### 1.3 共通の確認観点

- 画面がクラッシュしない
- 操作後にローディング状態が分かる
- 成功時に完了状態が分かる
- 失敗時にエラーが分かる
- 期待したデータが保存・反映される

---

## 2. AI記事生成

### 2.1 単発記事生成

**前提**
- AI設定が有効
- キーワードを1つ以上用意

**手順**
1. `AI記事生成` を開く
2. キーワードを入力する
3. 文体と文字数を選ぶ
4. `記事を生成` を押す

**期待結果**
- 生成中表示が出る
- 記事本文が表示される
- タイトル、本文、文字数が入る
- 記事一覧に保存される

**失敗時の確認**
- APIキー未設定
- モデル設定未選択
- レート制限
- 生成途中の UI 停止

### 2.2 再生成

**手順**
1. 生成済み記事を開く
2. 再生成機能を使う
3. 文字数や指示を変えて再実行する

**期待結果**
- 新しい内容で再生成される
- 目標文字数が大きく崩れない
- 以前の画面状態が壊れない

---

## 3. 対話型記事生成

### 3.1 Step1 トレンド分析

**手順**
1. 対話型記事生成を開く
2. キーワードを設定する
3. トレンド分析を実行する

**期待結果**
- 関連キーワードや競合情報が表示される
- エラー時は次ステップへ進まない

### 3.2 Step2 タイトル生成

**手順**
1. Step1 完了後にタイトル生成を実行する
2. 提案タイトルを確認する
3. 任意のタイトルを選ぶ

**期待結果**
- タイトル候補が複数出る
- 選択タイトルが次ステップへ引き継がれる

### 3.3 Step3 アウトライン生成・編集

**手順**
1. アウトライン生成を実行する
2. 既存見出しを編集する
3. 新しい見出しを追加する
4. 見出し順を入れ替える
5. 説明文や文字数を編集する

**期待結果**
- 見出し編集が画面に反映される
- 追加見出しが保持される
- 並び替え結果が保持される
- 次ステップへ進んでも編集内容が消えない

### 3.4 Step4 本文生成

**手順**
1. Step3 で見出しを1つ以上追加する
2. `本文の執筆を開始` を押す
3. 生成完了後、本文を確認する

**期待結果**
- Step3 で追加した見出しが本文に出る
- 見出し順が outline と一致する
- `まとめ` の自動補正で追加見出しが消えない

**重点確認**
- ユーザー追加見出しが欠落しない
- 自動追加のまとめが末尾に不正挿入されない

---

## 4. ファクトチェック

### 4.1 手動ファクトチェック

**前提**
- Perplexity API Key 設定済み

**手順**
1. 生成済み記事を開く
2. ファクトチェック実行ボタンを押す
3. 結果一覧を確認する

**期待結果**
- 進捗が表示される
- 判定結果が返る
- `correct / incorrect / partially_correct / unverified` が表示される

### 4.2 全文チェック確認

**手順**
1. 数値、日付、固有名詞を多く含む長文記事を用意する
2. ファクトチェックを実行する
3. 記事前半だけでなく後半の主張も結果に含まれるか確認する

**期待結果**
- 後半の主張も抽出対象になる
- 検証件数が少なすぎない

### 4.3 client モード強制有効化

**前提**
- `.env.local` に `VITE_DEPLOYMENT_MODE=client`

**手順**
1. 設定画面を開く
2. ファクトチェック設定を見る
3. client モードで記事生成する

**期待結果**
- 有効化トグルが表示されない、または常時有効表示になる
- client モードでは無効化できない
- 記事生成後に自動ファクトチェックが走る

### 4.4 自動修正

**手順**
1. 明らかに誤った数値や記述を含む記事を用意する
2. 自動修正 ON でファクトチェックを実行する
3. 修正後本文を確認する

**期待結果**
- 問題箇所が修正される
- 見出し構造が壊れない
- 記事全体が消えたり極端に短くなったりしない

## 5. WordPress連携

### 5.1 WordPress設定保存

**手順**
1. WordPress設定画面を開く
2. REST API URL、ユーザー名、アプリケーションパスワードを入力する
3. 保存する

**期待結果**
- 設定が保存される
- 再読込後も値が保持される

### 5.2 下書き保存

**手順**
1. 生成済み記事を開く
2. 公開状態を `draft` にする
3. WordPressへ送信する

**期待結果**
- WordPress 側に下書き記事が作成される
- タイトルと本文が一致する

### 5.3 公開

**手順**
1. 公開状態を `publish` にする
2. WordPressへ送信する

**期待結果**
- WordPress 側で公開状態になる
- 公開URLが取得できるなら確認する

### 5.4 予約投稿

**手順**
1. 公開状態を `future` にする
2. 未来日時を入力する
3. WordPressへ送信する

**期待結果**
- WordPress 側で予約投稿になる
- 予約日時が一致する

## 11. 回帰テスト最小セット

リリース前は最低でも以下を毎回実施します。

1. `npx tsc -b`
2. `npm run build`
3. 単発記事生成
4. 対話型記事生成
5. Step3 で見出し追加して Step4 実行
6. 手動ファクトチェック
7. client モードで自動ファクトチェック
8. WordPress 下書き保存
9. WordPress 公開
10. 記事一覧で編集・削除

---

## 12. 今のリポジトリで重点的に見るべき項目

- [ ] 対話型生成で追加見出しが本文へ確実に反映されるか
- [ ] `まとめ` セクション補正で末尾構成が壊れないか
- [ ] client モードでファクトチェックが自動実行されるか
- [ ] 自動修正後も本文と見出し構造が崩れないか
- [ ] `npm run build` が安定して通るか
