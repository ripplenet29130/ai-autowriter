# AutomaticWriter クライアント固有値一覧

このドキュメントは、外販時に「何を共通化し、何をクライアントごとの差分にするか」を固定するための一覧です。  
monorepo 化や `client-template` 作成前に、この表を埋めることを前提にします。

---

## 1. 使い方

1. 各項目について「共通」か「クライアント固有」かを決める
2. クライアント固有の場合、どこで管理するかを決める
3. 誰が設定するかを決める
4. `.env` で持つか、DB で持つか、app 側ファイルで持つかを固定する

---

## 2. 管理ルール

### 共通で持つべきもの

- 記事生成ロジック
- ファクトチェック本体
- WordPress連携ロジック
- スケジューラー基盤
- 共通UIの基本構造

### クライアント固有にしてよいもの

- アプリ名
- ロゴ
- 配色
- 接続先
- 通知先
- パスワード
- 機能表示の一部

## 4. 管理場所の候補

### `.env` に向いているもの

- 秘密情報
- 接続先
- deployment mode
- パスワード

### DB に向いているもの

- 運用中に変えたい設定
- 通知先
- ON/OFF 設定
- UI 上で変更させたい設定

### app 側ファイルに向いているもの

- ロゴ
- 色
- 固定文言
- ブランド名

- 自動公開を許可するか
