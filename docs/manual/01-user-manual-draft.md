# AutomaticWriter 利用マニュアル

## 1. このマニュアルの目的

このマニュアルは、AutomaticWriter を初めて使う人が、AI記事の生成から WordPress 投稿、自動投稿の設定まで進められるようにするための利用者向け手順書です。

## 2. 利用前に準備するもの

- AutomaticWriter にログインできるアカウント
- AIサービスのAPIキー
- 投稿先の WordPress サイト
- WordPress のユーザー名
- WordPress の Application Password
- 投稿先カテゴリーのID

## 3. 初期設定

### 3.1 AI設定

1. 左サイドバーから「AI設定」を開きます。
2. 使用するAIプロバイダーを選択します。
3. APIキーを入力します。
4. 「設定を保存」をクリックします。

保存後、AI設定はデータベースに保存されます。

### 3.2 WordPress設定

1. 左サイドバーから「WordPress設定」を開きます。
2. 「新しいWordPress設定を追加」をクリックします。
3. サイト名、WordPress URL、ユーザー名、Application Password、カテゴリーIDを入力します。
4. 「設定を保存」をクリックします。

Application Password は WordPress 管理画面の「ユーザー」から取得します。通常のログインパスワードではなく、Application Password を使用してください。

## 4. 記事を作成する

1. 記事生成画面を開きます。
2. キーワード、タイトル、記事の条件を入力します。
3. 記事を生成します。
4. 生成された本文を確認し、必要に応じて修正します。
5. WordPressへ投稿します。

## 5. 自動投稿を設定する

1. 左サイドバーから「スケジューラー」を開きます。
2. 対象の WordPress 設定を選びます。
3. 投稿頻度、実行時刻、投稿状態、ターゲットキーワードを設定します。
4. 設定を保存します。
5. 自動投稿を開始します。

ターゲットキーワードが未設定の場合、自動投稿は実行できません。

## 6. 動作確認

### 6.1 テスト投稿

1. スケジューラー画面でテスト用キーワードを入力します。
2. テスト生成を実行します。
3. 記事が生成され、WordPress に投稿されるか確認します。

### 6.2 投稿履歴の確認

投稿後は、AutomaticWriter の履歴画面または WordPress 側で投稿結果を確認します。

## 7. よくあるトラブル

### AI設定が保存できない

- APIキーが正しいか確認してください。
- 利用しているAIサービス側でキーが有効か確認してください。

### WordPressに投稿できない

- WordPress URL が正しいか確認してください。
- ユーザー名が正しいか確認してください。
- Application Password を使用しているか確認してください。
- カテゴリーIDが存在するか確認してください。

### 自動投稿が動かない

- AI設定が完了しているか確認してください。
- WordPress設定が有効になっているか確認してください。
- ターゲットキーワードが設定されているか確認してください。
- スケジュールが開始状態になっているか確認してください。
