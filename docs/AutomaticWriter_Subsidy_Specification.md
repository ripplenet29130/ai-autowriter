## 1. サービス概要と導入目的

「AutomaticWriter」は、企業が運営するオウンドメディアやブログ、コンテンツマーケティングにおける記事制作プロセスを、最新の生成AI（LLM）を用いて完全自動化するシステムです。

従来、担当者が手作業で行っていた「検索キーワード調査」「記事構成の作成」「執筆」「推敲」「ファクトチェック（事実確認）」「WordPressへの入稿指示」という一連の業務フローを統合・自動化することで、労働時間の大幅な削減と、コンテンツ品質の均一化（属人化の排除）を実現します。

**【本ツールが解決する企業の経営課題】**
- コンテンツ制作に関わる人件費・外注費の高騰
- 記事の品質が執筆者のスキルに依存してしまう（属人化）
- 記事制作に時間がかかり、Web集客のPDCAサイクルが遅れる

## 2. 主要機能・システム仕様

本システムは以下の主要機能を通じて、企業のデジタルトランスフォーメーション（DX）を支援します。

### ① マルチステップAI生成エンジン
キーワード入力から「タイトル案の生成」→「見出し構成（アウトライン）の生成」→「セクションごとの長文生成」→「最終的な文体統一と推敲」の4段階に分けてAI処理を実行します。これにより、単発のプロンプト入力では得られない高品質かつ長文のSEO記事を出力します。

### ② マルチLLM対応と最適なモデル選択
利用企業の意図に合わせて、OpenAI（GPTシリーズ）、Anthropic（Claudeシリーズ）、Google（Geminiシリーズ）のAPIを統合。処理ステップごとに最適なAIモデルを使い分けることが可能です。

### ③ 競合分析・トレンド反映機能
Web上の検索トレンドや、競合サイトの構成データをプログラムが自動で分析し、その結果（トレンドスコアや関連キーワード）をAIのプロンプト（指示書）に組み込みます。これにより、単なる一般論ではなく、最新情報に基づいた記事が生成されます。

### ④ 自動ファクトチェック（事実確認）機能
生成された記事に含まれる具体的な固有名詞や数値等に対して、AIが自動的にファクトチェックを実行します。人間によるダブルチェックの工数を大幅に削減し、信頼性の高い情報発信をサポートします。

### ⑤ コーポレートトーン（文体）の学習・適用
任意のURL（過去の自社ブログ記事など）を指定することで、システムがそのページの文体や表現スタイルを分析・参照し、自社のブランドトーンに合わせた自然な文章で出力します。

### ⑥ WordPress自動連携・スケジュール投稿機能
生成された記事は、企業のWordPressサイトへAPI経由で直接、自動投稿（または下書き保存）されます。「毎日 12:00」や「毎週月曜 09:00」といった無人スケジュール実行機能により、運用担当者の入稿作業がゼロになります。

## 3. 導入による効果（効率化・省力化のシミュレーション）

本システムの導入により、記事1本あたりの制作時間を**約90%以上削減**することができます。

**【導入前（従来の手作業フロー）】**
1記事（約3,000文字）あたりの想定作業時間：**約6.5時間**
- キーワード選定・競合調査： 1.0 時間
- 構成案（見出し）作成： 1.0 時間
- 本文の執筆： 3.0 時間
- 推敲・ファクトチェック： 1.0 時間
- WordPressでの装飾・入稿作業： 0.5 時間

**【導入後（AutomaticWriterによる自動化フロー）】**
1記事あたりの担当者作業時間：**約0.5時間（または完全無人化）**
- 配信スケジュールの設定： 約数分（初期のみ）
- 生成後の最終確認・公開承認： 約0.5時間
- （※スケジュール設定による自動実行の場合、作業時間は実質ゼロ）

**【経営数値へのインパクト（月間20記事を制作する場合）】**
- 導入前：6.5時間 × 20記事 = **130時間/月**の労働
- 導入後：0.5時間 × 20記事 = **10時間/月**の労働
⇒ **月間120時間の労働力削減（省力化）**を実現。削減されたリソースを、より付加価値の高い戦略立案や顧客対応にシフト（再配置）可能。

## 4. 対象ユーザー（導入推奨企業）

- オウンドメディアやSEOによる集客に注力している中小企業・小規模事業者
- コンテンツ制作を外部ライターに委託しており、外注費を削減したい企業
- 人的リソースが不足しており、Webマーケティング専任の担当者を配置できない企業
- 専門性の高い商材を扱っており、社内の知見（カタログ・マニュアル等）をベースにした記事を量産したい企業

## 5. システム構成と動作環境

SaaS（またはクラウド導入型システム）として提供され、利用企業側のサーバー構築等は不要です。

- **バックエンド/データベース**: Supabase (PostgreSQL), Edge Functions
- **フロントエンド UI**: React, TypeScript, Tailwind CSS
- **連携必須システム**:
  - WordPress (REST APIおよびアプリケーションパスワード機能を利用)
  - 各種AI API (OpenAI API, Anthropic API, Google Gemini API等)
- **ユーザー利用環境**:
  - 最新のGoogle Chrome / Microsoft Edgeブラウザを推奨
  - インターネット接続必須

## 6. ベンダーサポートとセキュリティ体制

- **初期導入支援**: 初期設定、AIプロンプトのカスタマイズサポート、WordPress連携設定などを含めたオンボーディングを提供。
- **データセキュリティ**: 入力データや生成コンテンツは、AIプロバイダの学習データとして使用されない仕様（各APIのオプトアウト設定準拠）で通信を行い、企業秘密や個人情報の漏洩リスクを最小化します。
- **保守・アップデート**: 検索エンジンのアルゴリズム変化や、AIモデルの進化に合わせたシステム側の継続アップデートを提供。
