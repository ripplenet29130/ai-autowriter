# 記事生成アプリの商品化に向けた要件整理

## 目的

現在の生成品質は、記事の初稿を短時間で作る用途としては十分に実用的である。
一方で、そのまま公開できる品質を毎回安定して出すには、生成機能だけでなく、表現のばらつきや軽微な不正確さを抑える仕組みが必要になる。

このドキュメントでは、AutomaticWriter を `人が最終調整する前提の実務向け記事作成支援アプリ` として商品レベルに引き上げるための要件を整理する。

## 前提評価

- 現状の品質は `80点前後の初稿を速く作る` 用途には向いている
- `完全自動公開` を前提にするには品質の安定性が不足している
- 商品価値は `より賢い生成` よりも `人の編集負荷をどこまで減らせるか` で決まる

## 想定する運用

- AI が記事の初稿を生成する
- 人が 5分から10分程度で最終調整を行う
- 人は主に `事実確認` `用語統一` `表現の自然さ` `CTA調整` を確認する
- 専門性の高いテーマは公開前レビューを必須とする

## 商品レベルに必要な中核要件

### 1. 品質安定化

- 用語統一チェック
  - 例: `クーラント` と `切削油` のような表記ゆれを自動検出して統一候補を出す
- 重複表現の検出
  - 同じ意味の説明が繰り返されている箇所を検知する
- 不自然表現の検出
  - 専門的に見えて曖昧な語、誤解を招きやすい表現、冗長な一文を警告する
- 文体の一貫性チェック
  - 敬体と常体の混在、BtoB向け記事での口語寄り表現などを抑える

### 2. 編集支援

- 見出し構成の自動整理
  - 導入、本文、まとめの流れを保ちながら、冗長な段落を圧縮する
- 文体切り替え
  - `やさしい` `専門的` `BtoB向け` `SEO向け` などを選択可能にする
- CTA の自然化
  - 問い合わせ導線を本文と自然につなげる
- 人が確認すべき箇所の明示
  - 危うい箇所を一覧化し、レビュー時間を短縮する

### 3. 実務運用への適合

- 業界別テンプレート
  - 製造業、不動産、士業、美容などで構成、語彙、注意点を切り替える
- 禁止表現チェック
  - 薬機法、景表法、断定表現、誇大表現などを警告する
- 事実確認アラート
  - 数値、比較表現、最新情報、根拠が必要な断定文を検知する
- 公開向け付帯要素の生成
  - タイトル、ディスクリプション、抜粋文などを同時に生成する
- WordPress 貼り付け向け整形
  - 見出しや段落の体裁をそのまま使える形に整える

## 最低限実装したい機能

- 見出し構成の自動整理
- 用語統一チェック
- 重複表現の検出
- 不自然表現チェック
- 文体切り替え
- CTA 自動調整

## 実務で強い機能

- 業界別テンプレート
- 禁止表現ルール
- 事実確認が必要な箇所の警告
- タイトルとディスクリプション生成
- WordPress 貼り付け向け整形

## 推奨する優先順位

1. 用語統一
2. 重複削減
3. 文体調整
4. 事実確認アラート
5. 業界別テンプレート
6. CTA 最適化

## MVP の定義

まず狙うべき MVP は、`完全自動公開ツール` ではなく `人の仕上げ時間を短縮する記事生成支援ツール` である。

MVP に必要な条件は以下。

- 記事の骨子が自然に出る
- 大きな破綻なく本文が展開される
- 表記ゆれと重複を自動検出できる
- 人が確認すべき箇所が分かる
- 最後に WordPress へ載せやすい形式に整えられる

## 評価指標

品質評価は感覚ではなく、最低限以下の観点で確認する。

- 正確性
- 自然さ
- 専門性
- SEO 適合
- CTA の自然さ
- 人の修正時間

## 結論

AutomaticWriter を商品レベルに近づけるうえで重要なのは、生成モデルをさらに複雑化することではない。
優先すべきなのは、`出力のばらつきを減らすこと` と `人が直すべき箇所をすぐ見つけられること` である。

この方向で機能追加を進めれば、単なる AI ライターではなく、現場で運用しやすい記事作成支援アプリとして成立しやすい。
