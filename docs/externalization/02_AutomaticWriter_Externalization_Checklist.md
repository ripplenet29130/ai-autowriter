# AutomaticWriter 外販前チェックリスト

このチェックリストは、AutomaticWriter をクライアント向けに外販する前に確認すべき項目を整理したものです。

## 1. リリース判定

- [ ] 見出し追加後に、追加した見出しが本文へ反映される
- [ ] 「まとめ」セクションの自動補正で、ユーザー追加見出しが消えない
- [ ] 対話型記事生成と自動生成の両方で記事生成が完了する
- [ ] ファクトチェックが client モードで常時有効になる
- [ ] ファクトチェック結果を記事へ反映できる
- [ ] WordPress への下書き保存が成功する
- [ ] WordPress への公開が成功する
- [ ] 設定未入力時に、致命的な画面崩れではなく分かるエラーになる
- [ ] `npx tsc -b` が通る
- [ ] `npm run build` が安定して通る

## 2. クライアント環境設定

- [ ] `.env.example` に必要な環境変数が揃っている
- [ ] 導入クライアントごとに `.env.local` / Vercel env で差し替えできる
- [ ] `VITE_DEPLOYMENT_MODE=client` を利用する前提が決まっている
- [x] Supabase URL / Key は当社管理の Supabase Project を導入クライアントごとに発行し、Vercel env に設定する
- [x] WordPress 接続先 URL と認証情報は投稿先サイトごとに登録・切り替えする
- [ ] 外部APIキーの管理者が誰か決まっている
- [ ] API利用料金の負担者が決まっている
- [x] Supabase 利用料は販売価格または保守費に含める

## 3. セキュリティ

- [ ] 外部公開URLに対する簡易パスワードゲートの有無を決めている
- [ ] 実運用ではパスワードゲートだけに依存しない方針にしている
- [ ] Supabase 側の RLS / 権限制御方針を決めている
- [ ] 設定変更者と運用者の権限を分ける必要があるか判断している
- [ ] APIキーを画面に平文で残さない
- [x] 導入クライアントごとのデータ分離方針を決めている

用語メモ:

- 導入クライアント: AutomaticWriter を契約・利用する HP 業者 / Web 制作会社 / Web 運用会社
- 投稿先サイト: 導入クライアントが管理する顧客の WordPress サイト
- WordPress 接続先は、導入クライアントごとの DB 内に投稿先サイト単位で複数登録する

Supabase 運用メモ:

- 本番外販では、導入クライアントごとに Supabase Project を分ける
- Supabase アカウント / Organization / 請求は当社側で管理する
- 当社が初期設定、Edge Functions、DB設定、環境変数連携、保守を行う
- Supabase 利用料は販売価格または保守費に含め、導入クライアントへ別請求しない

- [ ] 生成記事の最低品質基準を定義している

- [ ] 「AI生成結果は最終確認が必要」であることを明文化している

- [ ] 自動修正後に見出し構成が壊れていない
- [ ] 想定ジャンルごとのサンプル記事を複数本で検証している

- [ ] 「再試行で直る問題」と「設定修正が必要な問題」を切り分けられる
